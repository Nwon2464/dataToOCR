# v1.1 Searchable Scan PDF

## Goal

Create a PDF that preserves the original scanned page appearance while adding an invisible OCR text layer for search.

## Input

MinerU API output directory:

```text
data/mineru_api_output/uscpa_reg2_1_p022_024/

Main files used:

layout.json
*_origin.pdf
Script
scripts/export_searchable_scan_pdf.py
Example Command
python scripts/export_searchable_scan_pdf.py \
  --mineru-dir data/mineru_api_output/uscpa_reg2_1_p022_024 \
  --output exports/goodnotes/searchable_scan/uscpa_reg2_1_p022_024_searchable_cjk.pdf \
  --font-file /usr/share/fonts/opentype/noto/NotoSansCJK-Medium.ttc
Result

Succeeded:

Preserved original scanned page appearance
Added searchable OCR text layer
Japanese search worked
English search worked
CJK font embedding worked with Noto Sans CJK
Limitation

The output is searchable, but not reliably selectable.

GoodNotes could search Japanese and English terms, but text selection and drag behavior were inaccurate because the OCR text layer was inserted from span-level bounding boxes rather than true word-level or character-level text positioning.

Conclusion

v1.1 is useful as a searchable scan PDF exporter.

However, it is not suitable for the main goal of selecting/copying text inside GoodNotes.

The project will move to v1.2:

content_list_v2.json
→ content-first text study HTML / Markdown / normalized JSON
→ later PDF export
Status

Completed as an experiment.
Kept as a utility, but no longer the main development direction.
